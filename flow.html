BIDV Report AI Agent — System Architecture

LangGraph Pipeline · FastAPI · PostgreSQL · OpenAI GPT-4o · Docker

🏦 Tổng quan hệ thống

BIDV Report AI Agent là hệ thống tự động hóa việc soạn thảo báo cáo ngân hàng dựa trên AI. Người dùng upload một file .docx mẫu có các ô tô màu vàng đánh dấu vị trí cần điền, cùng với các tài liệu nguồn (báo cáo cũ, file số liệu). Hệ thống sẽ tự động phân tích, điền thông tin, cho phép người dùng review và xuất file hoàn chỉnh.

Pipeline được xây dựng trên LangGraph với cơ chế human-in-the-loop interrupt — AI xử lý tự động đến một điểm nhất định rồi dừng lại chờ con người phê duyệt trước khi xuất kết quả cuối cùng.

📋 Giải thích luồng xử lý

  • Bước 0 — Upload Template: Người dùng upload file .docx mẫu. Backend giải nén ZIP, đọc word/document.xml, tìm tất cả các run có thuộc tính w:highlight w:val="yellow" và lưu thành danh sách fields vào database.
  • Bước 1 — Upload tài liệu nguồn: Người dùng có thể upload nhiều file Word/PDF làm tài liệu tham khảo. Hệ thống dùng pandoc và pdftotext để trích xuất plain text.
  • Bước 2 — Chạy Pipeline: LangGraph khởi động, phân tích template, ghép toàn bộ text nguồn thành context cho AI (tối đa 14.000 ký tự/field).
  • Bước 3 — AI điền fields: GPT-4o được gọi lần lượt cho từng field với system prompt quy định văn phong hành chính ngân hàng. Mỗi field nhận được value, confidence (high/mid/low) và reason.
  • ⏸ Interrupt: LangGraph tự động dừng, frontend polling mỗi 2 giây để phát hiện trạng thái "reviewing" và hiển thị giao diện review.
  • Bước 4 — Human Review: Người dùng xem kết quả AI, chỉnh sửa nếu cần, nhấn Phê duyệt. Graph resume và lưu final_value.
  • Bước 5 — Export DOCX: Hệ thống load file template gốc từ DB, thay thế các ô vàng bằng giá trị đã duyệt trực tiếp trên XML, repack thành .docx mới và trả về browser.
⚠️ Lưu ý kỹ thuật: Việc thay thế nội dung thực hiện bằng str.replace() trực tiếp trên raw XML bytes thay vì dùng ElementTree.write() — để tránh mất namespace declarations khiến Word từ chối mở file.

⚙️ Lý do chọn từng công nghệ

  • LangGraph: Cho phép định nghĩa pipeline dưới dạng đồ thị có trạng thái, hỗ trợ interrupt_before để dừng giữa chừng chờ human review — không thể làm được với LangChain thông thường.
  • FastAPI + asyncpg: Async hoàn toàn, xử lý đồng thời nhiều pipeline mà không block. BackgroundTasks chạy LangGraph pipeline song song với API response.
  • OpenAI GPT-4o với JSON mode: Đảm bảo output luôn là JSON hợp lệ, không cần xử lý parsing lỗi. Phù hợp với structured output yêu cầu value + confidence + reason.
  • PostgreSQL + BYTEA: Lưu trực tiếp file .docx binary vào DB thay vì filesystem — đơn giản hóa backup, không cần quản lý file storage riêng.
  • zipfile + ElementTree (Python stdlib): Không cần cài thêm thư viện nặng như LibreOffice hay OpenPyXL, giảm Docker image size và tránh lỗi dependency.

🗄️ Thiết kế Database

  • templates: Lưu file .docx gốc dưới dạng BYTEA. Mỗi lần export sẽ đọc binary này để đảm bảo luôn dùng template gốc, không bị ảnh hưởng bởi các lần chạy trước.
  • template_fields: Danh sách các ô tô vàng được extract lúc upload, lưu para_idx (vị trí paragraph) và placeholder (text gốc) để mapping khi replace.
  • pipeline_runs: Mỗi lần chạy là một run độc lập. Cột langgraph_state JSONB lưu toàn bộ checkpoint state của LangGraph để có thể resume sau interrupt.
  • field_results: Lưu song song ai_value (gốc của AI) và final_value (sau khi human chỉnh sửa) — giúp audit trail và đánh giá chất lượng AI theo thời gian.

🐳 Deployment

Toàn bộ hệ thống chạy trong Docker Compose với 4 services trong cùng một internal network. Nginx đóng vai trò reverse proxy: request đến localhost:8080/api/* được forward sang FastAPI backend :8000, còn request tĩnh phục vụ frontend HTML từ /usr/share/nginx/html.

Volume pg_data persistent đảm bảo data không mất khi restart container. pgAdmin (:5050) cho phép xem và query database trực tiếp qua giao diện web mà không cần cài thêm tool.

Khởi chạy: docker compose up --build — hệ thống tự động chờ PostgreSQL healthy trước khi start backend nhờ depends_on: condition: service_healthy.
